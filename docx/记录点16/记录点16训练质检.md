### 异常分析

#### 1. 数值稳定性问题
- **Epoch 6**: Total Loss = -0.7224, Main Loss = -0.8625
- **Epoch 7**: Total Loss = -2.8727, Main Loss = -3.0117
- **问题**：在二分类交叉熵损失中，损失值应为正。负损失值表明可能存在：
  - 数值计算错误（如log(>1)导致负值）
  - 损失函数定义有问题
  - 梯度计算错误

#### 2. 训练监控不完整
- 日志中只有训练集指标，没有验证集指标
- 无法评估泛化性能和过拟合程度
- 训练集AUPR达到0.9737，可能存在严重过拟合

#### 3. Domain Loss变化异常
- Domain Loss从1.5119下降到1.3892，下降幅度很小
- 与其他损失项的变化趋势不一致

### 决策
**状态**：**失败**
**决策**：**回退重新设计**

### 问题总结
1. **数值稳定性**：Loss变为负值，表明损失计算或梯度传播存在根本问题
2. **监控缺失**：缺乏验证集指标，无法评估泛化性能
3. **潜在过拟合**：训练集AUPR过高（0.9737），可能过度拟合训练数据
4. **训练不完整**：只有7个epoch，且后期出现数值异常

### 建议给算法分析师
1. **检查损失函数**：验证交叉熵损失计算是否正确，避免log(>1)的情况
2. **添加验证监控**：在训练过程中加入验证集评估
3. **分析梯度**：检查梯度是否爆炸或消失
4. **增加训练轮数**：7个epoch可能不足，考虑增加训练轮数观察收敛行为
5. **检查数值稳定性**：添加梯度裁剪、损失值范围检查等机制

**根本原因推测**：可能是损失函数计算中的数值问题，或者模型输出概率值超出[0,1]范围导致log计算异常。
